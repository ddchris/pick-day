# 🛠️ 開發工具使用指南

## 📍 訪問開發工具

直接訪問：`http://localhost:3000/dev-tools`  
或透過選單：點擊 **🛠️ 開發工具**

---

## ✨ 功能說明

### 1. 📅 投票狀態控制 (新功能)
無需等待每月的 24-30 號，您可以直接強制修改投票狀態：
- **自動 (Auto)**: 使用系統時間判斷 (預設)。
- **強制開啟 (OPEN)**: 無視日期，強制開啟投票功能 (用於測試投票)。
- **強制關閉 (CLOSED)**: 無視日期，強制關閉投票 (用於測試截止後狀態)。

### 2. 🎲 模擬多用戶 (新功能)
想要測試多人投票？
- 點擊「**骰子 🎲 生成隨機用戶**」按鈕。
- 系統會自動生成一個隨機的 User ID 和名稱並自動登入。
- 您可以連續點擊生成多個用戶，每次回到首頁投票，即可模擬多人參與。

### 3. 快速預設
一鍵切換常用角色：
- **👑 管理員**: 測試管理員權限 (需在 .env 設定 NUXT_PUBLIC_ADMIN_USER_IDS)。
- **👤 一般用戶**: User 1。
- **👥 用戶 2**: User 2。
- **🗑️ 清除**: 登出。

## 🚀 測試場景範例

### 測試手動截止 (Manual Cut-off)
1. 在 `自動` 模式下，確認首頁顯示 (依日期而定)。
2. 在開發工具點擊 **「強制開啟 (OPEN)」**。
   - 前往首頁，確認可以投票。
   - 投幾票 (切換不同隨機用戶)。
3. 在開發工具點擊 **「強制關閉 (CLOSED)」**。
   - 前往首頁，確認：
     - 上方橫幅顯示「非填寫期間」。
     - 投票按鈕變灰無法點擊。
     - 應顯示投票統計 (O/X 數量)。

### 模擬 10 人投票
1. 點擊 **「🎲 生成隨機用戶」** -> 投票 O。
2. 點擊 **「🎲 生成隨機用戶」** -> 投票 X。
3. 重複 10 次。
4. 檢查首頁的計數是否正確增加。

## 📋 注意事項
- 狀態覆蓋僅在當前瀏覽器會話有效，重新整理後如果未清除 LocalStorage 可能會保留 (依實作而定，目前是非持久化)。
- 隨機用戶的 ID 是隨機生成的，不會與真實 LINE 帳號衝突。
